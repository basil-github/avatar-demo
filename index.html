Feedback System — Detailed Architecture
Step 1 — Binary Rating
👍 Good — answer correct & helpful
👎 Bad — answer missed or wrong
Collected via showcase webpage UI
Triggers immediate score update
Step 2 — Failure Type
Wrong information → Falcon issue
Incomplete answer → prompt issue
Wrong retrieval → FAISS issue
Voice/video issue → synthesis
Step 3 — Implicit Signals
Video replayed → soft +0.5 score
Followup asked → mild -0.2 score
Question repeated → -0.4 score
Watch time ≥ 80% → +0.3 score
Good feedback actions
Save Q&A to good_examples.json (few-shot pool, max 20)
Boost chunk FAISS score: scores[chunk_id] += 1.0
Queue for LoRA fine-tuning when pool ≥ 50 pairs
Stats dashboard: good_rate% trend updated
Bad feedback actions
Log to bad_answers.json with failure_type tag
Penalise chunk score: scores[chunk_id] -= 1.0
Trigger retry: re-prompt Falcon with expanded query
If wrong_retrieval → query expansion
Footnote Extraction — الطبيب الشعبي Document Format
Document structure
Layer 1 — Section heading
OCR & Text Extract: pdfplumber → pytesseract, lang='ara', psm=6
Layer 2 — Body text with superscript refs
الفَقَّارَه يَمْسَحُونَها ²³⁶ وَعَصَبِيَة — numbers 236/237/238 are meaningless to FAISS without resolution
Layer 3 — Footnote definitions
236 يهمزونها: تدليكها بالمساج · 237 ولا: أو · 238 حُمَّله: أداة خشبية

1
OCR & Text Extract

2
Detect Headings
regex: ^[\u0600-\u06FF]{3,30}:$
3
Extract Footnotes
regex: (\d{3})\s+([\u0600-\u06FF]+)
4
Replace Inline Refs
re.sub(r'\b236\b', '[تدليكها]')
5
Build Enriched Chunk
heading + enriched_body + footnotes

About

This system combines RAG with text-to-speech voice cloning and video lip-sync to create an interactive knowledge interface. The source material is a book on Emirati folk medicine (الطبيب الشعبي), documenting traditional healing practices. The avatar serves as a digital narrator, preserving this cultural knowledge in an engaging format.
